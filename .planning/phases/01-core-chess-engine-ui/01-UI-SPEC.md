# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Core Chess Engine + UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla HTML5 + CSS custom properties |
| Icon library | none — Unicode box-drawing characters + chess glyphs |
| Font | monospace system stack: `"Courier New", Courier, monospace` |

**Rationale:** Vanilla HTML/CSS/JS frontend per tech stack decision. No React/Next.js/Vite — shadcn gate not applicable. TUI aesthetic requires monospace throughout. No external component library.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | CSS Custom Property | Value | Usage |
|-------|---------------------|-------|-------|
| xs | `--sp-xs` | 4px | Icon gaps, inline glyph padding, border-char alignment |
| sm | `--sp-sm` | 8px | Cell padding inside board squares, compact element spacing |
| md | `--sp-md` | 16px | Panel internal padding, command input vertical padding |
| lg | `--sp-lg` | 24px | Gap between board and right panel |
| xl | `--sp-xl` | 32px | Major layout column gap |
| 2xl | `--sp-2xl` | 48px | Page-level vertical margin |
| 3xl | `--sp-3xl` | 64px | Reserved for future phase expansion |

**Exceptions:**
- Touch / TV remote focus ring: minimum 44px hit area on panel toggle buttons (keyboard navigation only — no pointer required, but enforced for on-screen virtual keyboard compatibility).
- Board squares: size is computed dynamically via CSS `--board-font-size` variable, not from the spacing scale. See Typography section.

---

## Typography

All text uses the monospace stack. No serif or sans-serif fonts anywhere.

| Role | Size | CSS Custom Property | Weight | Line Height | Usage |
|------|------|---------------------|--------|-------------|-------|
| Body | 16px | `--font-size-body` | 400 | 1.5 | Panel text, move history entries, input prompt |
| Label | 14px | `--font-size-label` | 400 | 1.4 | Panel headers, column labels (rank/file identifiers a–h, 1–8), config menu option labels |
| Heading | 20px | `--font-size-heading` | 600 | 1.2 | Screen-level titles ("New Game", pre-game config screen title), section headers |
| Display | 28px | `--font-size-display` | 600 | 1.1 | Game outcome announcement ("White wins", "Draw") |

**Board piece font size:** Controlled exclusively by `--board-font-size` CSS custom property (default: 24px). This is independent of the typography scale and scales for TV distance readability. Users can adjust via settings.

**Font weight constraint:** Only two weights used: 400 (regular) and 600 (semibold). No bold (700), no light (300).

---

## Color

Terminal dark aesthetic. No gradients. No shadows. No animations.

| Role | CSS Custom Property | Value | Usage |
|------|---------------------|-------|-------|
| Dominant (60%) | `--color-bg` | `#0d0d0d` | Page background, board dark squares, collapsed panel backgrounds |
| Secondary (30%) | `--color-surface` | `#1a1a1a` | Panel backgrounds (move history, PGN list, command input area), board light squares |
| Accent (10%) | `--color-accent` | `#c8a84b` | Reserved — see list below |
| Destructive | `--color-destructive` | `#e05252` | Resign confirmation text only |
| Text primary | `--color-text` | `#e8e8e8` | All body and label text |
| Text muted | `--color-text-muted` | `#666666` | Disabled options, rank/file coordinate labels |
| Border | `--color-border` | `#333333` | ASCII box-drawing panel borders (`┌─┐│└┘`) |
| Focus | `--color-focus` | `#c8a84b` | Keyboard focus ring on panel toggles and config menu items |

**Accent (`#c8a84b` — warm gold) reserved for:**
1. Active/selected square highlight on board (last move origin + destination)
2. Focused option in pre-game config menu cursor (`>` indicator)
3. Command input prompt cursor character
4. Selected move in move history panel

**Accent is NOT used for:** general hover states, decorative borders, loading indicators.

## Component Inventory

These are the UI components this phase must implement. All are custom-built (no library), using ASCII box-drawing characters for borders.

### 1. Chess Board (`#board`)
- 8×8 grid rendered as a `<table>` or CSS grid
- Dark squares: `--color-bg`; Light squares: `--color-surface`
- Pieces rendered as text: Unicode default (`♔ ♕ ♖ ♗ ♘ ♙ / ♚ ♛ ♜ ♝ ♞ ♟`) or ASCII (`K Q R B N P / k q r b n p`)
- Font size: `--board-font-size` (default 24px, user-adjustable)
- Last move squares highlighted with `--color-accent` background tint (`#c8a84b22`)
- Rank labels (1–8) left of board, file labels (a–h) below board, both in `--color-text-muted`, `--font-size-label`
- Board is left-aligned in landscape layout, fills ~55% of viewport width

### 2. Command Input (`#cmd-input`)
- Always visible at bottom of right panel — never hidden, never collapsed
- Single-line `<input type="text">` with monospace font
- Prompt prefix: `> ` in `--color-accent`
- Input text: `--color-text`, `--color-surface` background
- Border: 1px `--color-border` top only (underline style)
- Placeholder: `e4, Nf3, O-O...` in `--color-text-muted`
- On Enter: submit move; clear input
- Error feedback: input border turns `--color-destructive` for 1 frame (CSS transition: none — TUI feel)

### 3. Move History Panel (`#panel-history`)
- Collapsible via keyboard (arrow key or `H` shortcut)
- Header: `┌─ Move History ─┐` box-drawing border, `--font-size-label`, `--color-text-muted`
- Entries: `1. e4 e5`, `2. Nf3 Nc6` — monospace tabular alignment
- Selected entry highlighted with `--color-accent` foreground + `>` prefix
- Scroll: CSS `overflow-y: auto` within fixed panel height

### 4. PGN List Panel (`#panel-pgn`)
- Collapsible via keyboard
- Displays full game PGN string as it builds
- Header: `┌─ PGN ─┐`
- Monospace text, selectable for copy
- Same visual pattern as Move History Panel

### 5. Material Advantage Panel (`#panel-material`)
- Collapsible via keyboard
- Shows captured pieces grouped by type: `♟♟♟ +3` (white advantage) or `♙♙ -2`
- One line per player
- Header: `┌─ Material ─┐`

### 6. Pre-Game Config Menu (`#config-menu`)
- Full-screen overlay (replaces board view before game starts)
- Dark background `--color-bg`, monospace font throughout
- Title: "New Game" in `--font-size-heading`, `--color-text`, centered
- Option groups laid out horizontally per row, separated by spaces
- Current value shown as: `← Embedded →` (arrow chars `←` `→` flank selected value)
- Numeric controls: `- 3 +` pattern
- Focused option: `> Option Name: value` — `>` cursor in `--color-accent`
- Navigation: Arrow keys move focus; Enter confirms; Escape cancels (returns to last game or shows empty board)
- Options in order: Engine, Time Control, Color, Bot Difficulty, Tutor Mode, Physical Board Mode

### 7. Panel Toggle / Hamburger (`#panel-toggle`)
- Single button top-right of right panel
- Label: `[≡]` — monospace, `--color-text`
- On activate: opens mega-menu listing all collapsible panels with toggle state
- Keyboard: Tab to reach; Enter/Space to activate; Escape to close
- Min hit area: 44×44px

### 8. Physical Board Sync Indicator (`#sync-indicator`)
- Visible only when Physical Board Mode is active
- Displays: `[ WAITING — press SPACE to continue ]` centered below board
- Background: `--color-surface` bar spanning full board width
- Text: `--color-accent`, `--font-size-label`
- Hidden when game is running normally

│            CHESS BOARD                   │  │ 1. e4  e5       │   │
│            (55% viewport width)          │  │ 2. Nf3 Nc6      │   │
│                                          │  └─────────────────┘   │
│                                          │  ┌─ PGN ─┐            │
│                                          │  │ 1.e4 e5 2.Nf3 │    │

│                                          │  ┌─ Material ─┐        │
│                                          │  │ ♟♟ +2       │        │

- Two-column layout: board left (~55%), panels right (~45%)
- Minimum viewport: 768px wide (landscape constraint — REQ-003)
- Right panel: flex column, panels stack vertically, command input pinned to bottom
- No portrait support — display a `rotate device` message if `orientation: portrait` detected

## Interaction Contract

### Keyboard Navigation Map

| Key | Context | Action |
|-----|---------|--------|
| Any printable key | Game active | Focuses command input if not already focused |
| Enter | Command input focused | Submit move |
| Escape | Command input focused | Clear input without submitting |
| Space | Physical Board Mode, waiting state | Resume game after engine move |
| Arrow Up/Down | Config menu | Move focus between options |
| Arrow Left/Right | Config menu | Cycle option values |
| Enter | Config menu | Confirm selection / start game |
| Escape | Config menu | Cancel / return to game |
| H | Game active | Toggle Move History panel |
| P | Game active | Toggle PGN panel |
| M | Game active | Toggle Material panel |
| Tab | Game active | Cycle focus: Command Input → Panel Toggle → back |

### States

| Component | States |
|-----------|--------|
| Command Input | default, focused, error (invalid move — 1-frame border color) |
| Board Square | default, last-move-origin, last-move-destination, check-king |
| Panel | collapsed, expanded |
| Config Option | default, focused (cursor `>`) |
| Physical Sync Indicator | hidden, visible-waiting |
| Game | pre-config, playing, paused (physical mode), ended |

### Check State
- King square in check: background `--color-destructive` tint (`#e0525222`)
- No animation — static highlight

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (config menu) | `Start Game` |
| Command input placeholder | `e4, Nf3, O-O...` |
| Invalid move error | `Invalid move — try again` (below input, `--color-destructive`, clears after next keypress) |
| Empty move history | `No moves yet` (muted, center-aligned in panel) |
| Empty PGN panel | `Game not started` |
| Physical sync prompt | `WAITING — press SPACE to continue` |
| Game over — checkmate | `Checkmate. {Color} wins.` |
| Game over — draw | `Draw.` |
| Game over — resignation | `{Color} resigned.` |
| Resign confirmation | `Resign? Press R again to confirm.` |
| Config menu title | `New Game` |
| Config: Engine option | `Engine:` |
| Config: Time Control option | `Time Control:` |
| Config: Color option | `Play as:` |
| Config: Difficulty option | `Difficulty:` |
| Config: Tutor Mode option | `Tutor Mode:` |
| Config: Physical Board option | `Physical Board:` |
| Rotate device message | `Rotate device to landscape to play.` |

**Destructive actions in Phase 1:** Resign only. Confirmation: press `R` twice (first press shows inline confirmation copy; second press within 3 seconds executes). No modal dialog — inline text below command input.
